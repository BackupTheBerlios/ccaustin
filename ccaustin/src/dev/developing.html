Developing Pages

This page provides an overview of the process, components, and roles of the Calvary 2.0 Web site for understanding how everything fits together to compose the static web pages. It also provides a simple step-by-step guide for building your first few XML/XSL pages after you have successfully installed and processed existing pages into HTML.

Components

XML Document

Contains the data for one or more targeted HTML pages. Data is usually in the format:

<content title="Our Vision">
	<visionstatement>
		<purpose>To win lost people to Christ,
       to equip them for ministry, and to send 
       them to serve. 
       </purpose>
       <vision>
			<item id="1" goal="Win a person to Christ"
            goalLogo="../images/win.gif" 
            goalImg="../images/winttl.gif"
            according="Mark 16:15" 
            quote="and He said to them,..."/>
       </vision>
   </visionstatement>
XSL Document

An XSL Document uses an XML format and special tags to tell an XSL processor how to convert XML to any format. For the web site, we use HTML so the XSL file will contain HTML templates with XSL code that manipulates an XML document's contents and merges them into the resulting HTML. XSL has a very simple format and is easy to learn. XSL usually is in the format:

<?xml version="1.0"?>
    
<xsl:stylesheet version="1.0" 
 xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

 <xsl:output method="html" indent="yes"
  doctype-system="http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"
  doctype-public="-//W3C//DTD XHTML 1.0 Transitional//EN" />

 <xsl:include href="../general-ui.xsl"/>
 <xsl:include href="../general-html.xsl"/>
 
 <xsl:template match="visionstatement">
   <xsl:apply-templates select="purpose"/>
   <xsl:apply-templates select="vision"/>
     <p align="center"><xsl:apply-templates select="text"/></p>
 </xsl:template>

</xsl:stylesheet>
Ant Script

Ant is a pure Java version of Make and uses XML to define the steps necessary to compile code, process XML and XSL, and perform other tasks like creating, removing, and renaming files and folders. It in essence provides a simple way to manage project work without writing a lot of hard to read scripting. The script is used by Calvary Web to perform XSL transformation by identifying the XML, XSL and resulting HTML output names and it will do the appropriate work using the Apache Xalan XSL processor.

HTML

The HTML is a result of the XSL + XML processed through the Ant script.

Roles

The following roles are required for the Calvary Web site to be created and published:

Content Writer
The content writer is responsible for working with ministry leaders to obtain the appropriate ministry information, page requirements, and content. This is then used to construct XML documents with the data, and work with the HTML writer to determine the best method of page structure and layout. The content writer need only to know how to use the most basic of text editors (notepad or similiar) to do their job, and must be willing to interact with a number of ministries and stay abreast of changes in ministry events and update the content appropriately.
HTML Writer

The HTML writer role requires basic knowledge of HTML and possibly Javascript. They should have a flare for web design and be able to work with content writers on the appropriate page structure and layout. They may utilize others familiar with web graphics design, or if possible, construct the web graphics themselves. The HTML writer works with the XSL writer to help transition the design of web pages into the appropriate format as the XSL is constructed (no XSL expertise is required).

XSL Writer

The XSL writer role requires a bit more of a scripting or programming background, as they must be able to use XML and HTML sources to properly construct an XSL document. Control flow, such as loops, tests, and text/data maniplation is common in XSL to properly process an XML document and create HTML. HTML background is nice but not a must, since buggy XSL templates could render bad HTML and cause problems that must be debugged.

User with Browser

The user and browser is the target audience and overall recipient of the hard work. Prior to a user being able to view a page, the page should be reviewed by one or more ministry staff and then published from the local Calvary Austin network to the Costa Mesa servers.
