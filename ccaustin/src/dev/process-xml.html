XML Document

An XML document is a text file that contains data, marked up by a beginning and ending tag set. Generally, an XML document should be thought of as content, or data, which is different from HTML pages which contain both content and layout (how it should look). An XML document can be edited using any sort of standard text editor, such as Wordpad that comes with Windows, or a full text editor like Emacs. It should never be edited with something like Microsoft Office, as it will ruin the content.

An XML document is considered well-formed if all of its tags are properly opened and closed and it has the appropriate XML declaration heading in it. If a document is not well-formed, it cannot be processed and therefore cannot be published. The following is an example of a well-formed XML document:

<?xml version="1.0" encoding="UTF-8"?>
<content title="About Us">
	<heading>10th Anniversary in Austin</heading>
	<p>Sunday, June 2 will mark two milestones
	  for Calvary Chapel of Austin: 10 years of 
      meeting as a fellowship, and 4 years in 
      our current building. 
    </p>
</content>

The following is not a well-formed document (notice the end tags </p> and </content> are missing:

<?xml version="1.0" encoding="UTF-8"?>
<content title="About Us">
	<heading>10th Anniversary in Austin</heading>
	<p>Sunday
