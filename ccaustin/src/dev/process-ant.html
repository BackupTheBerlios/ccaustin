Ant Script

What's an Ant script? Well, its a library from the Apache group (they make web server software) which allows you to define the steps needed to build a project. We use Ant, which requires Java 1.3.1 or newer, to define how our web site gets built - from the XSL transformations, date/version substitutions, staging deployment, and publishing to production. An Ant script has one or more targets, which are commands that tell the script to do things. We currently use the following targets:

Web target

The web target builds the entire web site by using the sitemap.xml as a list of pages to build. This enables you to see the web site changes you made on your local machine, or build the site from the latest code stored in CVS source control.

Dist target

The dist target copies the web site, all associated images, and supporting files (stylesheets, javascript libraries, etc) to a specific directory. This enables us to stage the site for our hourly builds (done automatically).

Publish target

The publish target takes a distribution and FTPs it to the production server, using the id and password located in a local.properties file you create. This is done automatically, but also could be done manually from our staging server in an emergency.
